## 📚 文档一致性问题

### 1. skill-management-tools-design.md 与实际实现差异

**设计文档路径**：`docs/design/v2/skill-management-tools-design.md`

**文档描述**：
- 设计了三個内置工具：`list_skills`, `save_skill`, `delete_skill`
- 用于 AI 专家通过对话管理技能

**实际实现**：
- 实际 API 路由在 `server/routes/skill.routes.js`
- 提供 RESTful API 而非内置工具
- 缺少 `save_skill` 工具的实现（只有 `register`, `assign`, `unassign` 等）

**差异分析**：
1. **架构差异**：设计文档规划的是"内置工具"，实际实现的是 REST API
2. **功能缺失**：设计中的 `save_skill` 工具未找到对应实现
3. **命名不一致**：设计中用 `list_skills`，实际 API 是 `GET /api/skills`

**建议**：
- 更新设计文档，标记为"已变更"
- 或实现设计中规划的内置工具集

**优先级**：🟡 **P1 - 文档更新**
