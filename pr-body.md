## 问题描述

`GET /api/skills/:id/parameters` 接口只需要 `authenticate()` 认证，任何登录用户都能查看技能参数值。

### 安全风险
- 普通用户可以看到全局参数值，包括敏感参数（如 API Key）
- 参数值可能包含数据库连接字符串、第三方服务密钥等敏感信息

## 解决方案

### 后端修改
- 将 `GET /api/skills/:id/parameters` 接口改为管理员权限

### 前端修改
- 「管理参数」按钮只对管理员显示
- 「启用/禁用」按钮只对管理员显示

## 测试要点

1. 普通用户登录后，技能卡片不显示「管理参数」和「启用/禁用」按钮
2. 普通用户直接访问 `/api/skills/:id/parameters` 返回 403 错误
3. 管理员可以正常查看和管理参数
